4. DIAGNÓSTICO DE LOS 5 FALLOS
-------------------------------
Todas las cuentas con HTTP 500 significa que:
1. La API estaba activa (code=500, no 300/404)
2. MySQL deadlock en el servidor de UNCP
3. El nuevo engine les dio 3 oportunidades (bursts 1, 2, 3)
4. Las 3 veces dieron 500 → la cuenta probablemente está dañada o el
   deadlock fue persistente para ese DNI específico

Posibles causas de 500 persistente:
- DNI expirado/no matriculado que causa error en MySQL al validar
- Cuenta bloqueada que genera deadlock en cada intento
- Certificate/ID problem que hace que MySQL falle consistentemente

Solución: trackear estos DNIs día a día. Si fallan 3+ días seguidos con 500 → skip.
